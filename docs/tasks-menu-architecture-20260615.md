## 0. 执行边界

本次任务只调整菜单结构、菜单显示、菜单角色授权。

允许改动：

```text
sys_menu.menu_name
sys_menu.parent_id
sys_menu.order_num
sys_menu.visible
sys_menu.status
sys_role_menu
必要的前端菜单标题/面包屑文案
```

不允许改动：

```text
报价单接入业务逻辑
价格类型确认逻辑
价格准备逻辑
成本核算逻辑
自制件价格生成逻辑
包装件/包装组件逻辑
BOM 结算逻辑
CMS/U9 数据读取逻辑
接口入参和响应结构
数据库业务表结构
```

隐藏菜单不等于删除功能。流程内部页面仍可被报价单接入详情页调用。

## 1. 执行规则

1. 任务必须按 T00 → T10 顺序执行。
2. 每个任务完成后必须留下证据：JSON 快照、截图、SQL 文件或测试记录。
3. 每个写操作前必须先做快照。
4. 不删除旧菜单记录，优先通过 `visible = 1` 隐藏，便于回滚。
5. 不修改菜单 `component`，除非只是为了修正明显错误的组件路径。
6. 不改接口权限 `perms`，除非角色菜单绑定验证明确需要补充。
7. 如果普通财务无法从 `报价单接入` 完成价格类型、价格准备、成本核算，立即回滚该阶段。
